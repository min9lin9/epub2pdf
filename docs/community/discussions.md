# GitHub Discussions Onboarding

GitHub Discussions is the main place for questions, ideas, announcements, and informal community chat. Issues are reserved for bug reports and concrete tasks.

## Welcome thread

Start with the pinned welcome thread:

- [👋 Welcome! Read this before posting](https://github.com/min9lin9/epub2pdf/discussions/13)

It explains where to post, what to include when asking for help, and links to good first issues.

## Discussion categories

| Category | Use it for | Example |
|---|---|---|
| **General** | Announcements, introductions, open-ended conversation | "Hello from a new contributor" |
| **Q&A** | Questions with a specific answer | "How do I install the WeasyPrint backend on macOS?" |
| **Ideas** | Feature requests and design proposals before they become issues | "Could we add a `--quiet` flag?" |
| **Show and tell** | Projects, scripts, or integrations built with epub2pdf | "I wrote a batch script for my library" |

## Before you post

1. Search existing discussions and issues for your question.
2. Read [docs/troubleshooting.md](../troubleshooting.md) if your question is about installation or runtime errors.
3. Include useful context: OS, Python version, installed extras, and the command you ran (with `--verbose` output if possible).

## From discussion to issue

If a discussion produces a concrete, actionable item, a maintainer will open an issue and link back to the discussion. You are also welcome to open an issue yourself once the scope is clear.

## Moderation

Discussions follow the same [CODE_OF_CONDUCT.md](../../CODE_OF_CONDUCT.md) as issues and pull requests. Maintainers may move off-topic posts to a more appropriate category or ask for clarification.
