# Code of Conduct

This project adheres to the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) as a baseline. The sections below add project-specific guidance.

## Our standards

We strive to create a welcoming, inclusive environment. Examples of expected behavior:

- Using welcoming and inclusive language.
- Being respectful of differing viewpoints and experiences.
- Gracefully accepting constructive criticism.
- Focusing on what is best for the community and users.
- Assuming good intent, especially from new contributors.

## Unacceptable behavior

- Harassment, trolling, or personal attacks.
- Sustained disruption of discussions or review threads.
- Publishing private information about others without permission.
- Spamming issues or pull requests.
- Dismissing contributors based on background or experience.

## Technical criticism vs. personal attacks

Disagreeing with a design decision, pointing out a bug, or questioning a dependency choice is welcome. Attacking the person who proposed it is not.

Good: "This renderer change may break EPUBs with fixed-layout CSS because ..."
Not acceptable: "You clearly don't understand how CSS works."

## Repeated or spam PRs

Pull requests that:

- are clearly automated or generated in bulk without human review,
- make no meaningful change,
- reopen closed discussions without new information, or
- repeatedly bypass the PR template

may be closed without extensive discussion. Maintainers will leave a short explanation.

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainers via GitHub or email. If the issue involves a maintainer, contact the Project Lead directly.
